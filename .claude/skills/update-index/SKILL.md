# Update Index

根据 Git 变更文件，自下而上检查并更新项目中的索引文件。

## Trigger

用户手动调用："更新索引" / "update index" / "同步索引文件"

## Instructions

### 1. 获取变更文件

Run:
- git status --porcelain
- git diff --name-only
- git ls-files --others --exclude-standard

合并去重，得到所有新增/修改/删除/重命名的文件。

### 2. 过滤排除项

排除：
- 配置文件（.json, .yaml, .yml, .toml, .ini, .conf）
- 锁文件（uv.lock, package-lock.json 等）
- 二进制文件（图片、压缩包等）
- .claude/ 目录下的文件

### 3. 确定起点目录

对每个变更文件，取其直接所在目录作为起点。

### 4. 自下而上遍历

从起点目录开始，逐级向上至项目根目录：

1. 扫描当前目录下的索引文件（不区分大小写）：
   - CLAUDE.md
   - index.md
   - readme.md / README.md
   - summary.md
2. 对比索引内容与实际目录结构，判断是否需要更新
3. 记录需要更新的文件及建议内容
4. 移动到父目录，重复直到根目录

### 5. 汇总方案

向用户展示待更新清单：

```
以下索引文件需要更新：

1. [文件路径]
   - 建议更新内容：[摘要]
   - 原因：[新增/删除/重命名了什么]
```

等待用户确认。

### 6. 执行更新

- 用户确认后逐文件更新
- 保持原文件格式风格
- 更新索引文件不触发新的更新循环

## Constraints

- 更新前必须展示方案并等待确认
- 同目录多个索引文件都需检查
- 不触发递归更新循环
